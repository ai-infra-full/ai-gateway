# ai-gateway

使用Axum框架做AI网关。

## 协议适配与模型接入（接入层）

* 多模型统一抽象（Unified API）： * 将所有上游模型（OpenAI, Claude, Gemini, DeepSeek, 百度文心等）统一抽象为标准格式（通常对齐 OpenAI 规范）。
    * 协议转换引擎（Protocol Translation）： 客户端用一套通用JSON格式请求，网关自动将其转换为目标厂商的特有Payload。

* 多模态与流式传输支持：
    * 原生支持 Server-Sent Events (SSE) 流式响应（Stream），并具备流式数据的拦截与解析能力。
    * 支持多模态输入（文本、图片 Base64、音视频流、PDF 等文档）的解析与分发。

* 生态协议接入（MCP 支持）：
    * 集成 MCP（Model Context Protocol，模型上下文协议），允许网关直接代理或接入外部 Agent 服务与远程工具（Tools）。

## 流量治理与高可用（调度层）

大模型 API 经常遇到限流（TPM/RPM 限制）和单点故障，网关必须具备智能调度能力。

* 智能路由与负载均衡（Smart Routing）：
    * 基于降本/低延时路由： 根据 Prompt 的复杂度或业务标签，简单的任务路由给轻量便宜的模型，复杂的任务路由给旗舰模型。
    * 动态权重与轮询： 跨不同供应商或混合云（公有云 API + 私有化部署大模型）进行流量负载均衡。
* 弹性容错机制（Failover & Resiliency）：
    * 自动降级（Fallback）： 当调用 GPT-4o 报错或触发 429 Too Many Requests 时，网关在毫秒级内自动无缝切换到 Claude 3.5 或 DeepSeek，对前端用户完全透明。
    * 智能重试与熔断： 针对特定的 LLM 错误码（如超时、服务过载）实施指数退避重试；当某模型彻底挂掉时自动熔断。

## 全链路安全与合规（安全层）

AI 场景带来了全新的安全挑战，如提示词注入、数据外泄、不合规言论等。

* 双向内容审计与内容安全（Guardrails）：
    * 输入审查（Prompt Guard）： 拦截敏感词、政治/暴力言论，防范提示词注入攻击（Prompt Injection）。
    * 输出审查（Output Moderation）： 实时检测模型生成的内容是否包含幻觉、违法或攻击性言论，支持流式文本的实时截断或替换。

* 敏感数据脱敏（Data Masking）：
    * 在请求发送给公有云大模型前，自动识别并脱敏手机号、身份证、企业机密代码、API Key 等敏感隐私数据（PII），在响应返回时进行复原。

* 企业级鉴权与多租户：
    * 统一的 API Key / OAuth2 认证。
    * 多租户（团队/项目/应用）权限隔离，防止 Key 被盗刷或滥用。

## 成本控制与可观测性（运营层）

大模型是按 Token 计费的，成本和性能的精细化监控是 AI 网关的刚需。
* Token 级别的精细化限流（Rate Limiting）：
    * 传统的网关只能限制“每分钟请求数（RPM）”。AI 网关必须支持 TPM（每分钟 Token 数） 和 RPD（每日请求数） 的多维度限流。

* 财务与成本计量（Billing & Quota）：
    * 按团队/项目实时统计 Token 消耗量，并根据各厂商最新币种价格自动换算为企业内部的“账单”。
    * 支持预算硬上限（Hard Caps），某个项目本月额度用完自动熔断拦截。

* AI专有指标监控（Observability）：
    * 支持对TTFT（Time to First Token，首字延迟）、推理吞吐量（Tokens/sec）、模型错误率、缓存命中率进行多维监控，并提供可视化 Dashboard。

* 提示词/全链路审计日志（Audit Logs）：
    * 完整记录“User Prompt -> Model Response”的交互轨迹，用于合规审计及后续的模型微调（Fine-tuning）数据收集。

## 智能增强扩展（应用层/性能优化）

在网关层叠加一层“轻量级智能”，能大幅提升系统性能。

* 语义缓存（Semantic Cache）：
    * 传统缓存必须字符串完全一致。AI 网关通过集成向量数据库，对 Prompt 进行向量化（Embedding）匹配。如果当前问题与历史问题的语义相似度超过 95%（如“帮我写个请假条”和“如何写一份请假申请”），直接返回缓存的答案。
    * 效果： 响应延迟从数秒骤降至毫秒级，且 Token 成本为 0。

* 提示词管理与模版化（Prompt Management）：
    * 将 Prompt 模版托管在网关层（如定义好 {"template_id": "translate_en", "text": "..."}）。
    * 支持网关层的提示词注入/装饰（Prompt Decorator），在发送给模型前自动追加系统设定（System Prompt）或安全上下文。

* 轻量级 RAG 组件/上下文注入：
    * 支持在网关层对接企业内部的知识库向量端点，在 Prompt 转发前动态检索并注入相关的企业背景上下文。
